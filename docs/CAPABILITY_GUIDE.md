# 能力调用指引

这个文档用来登记 ProjectPilot 会调用的飞书能力、LLM 能力和本地运行能力。后续新增服务或 Skill 时，先在这里补齐能力边界，再写代码。

## 能力分层

| 类型 | 作用 | 当前优先级 |
| --- | --- | --- |
| Wiki / Docs | 创建和更新项目知识库、读取源文档 | P0 |
| Base | 项目作战室、任务池、风险表、看板和仪表盘 | P0 |
| IM | 群聊触发、确认、主动分发 | P0 |
| Task | 创建待办、同步状态、负责人闭环 | P0 |
| Calendar / VC / Minutes | 会议输入、会后纪要和 Action Items | P0/P1 |
| Contact | 识别负责人和成员身份 | P1 |
| Event | 主动监听和低频触发 | P1 |
| LLM | 解析、总结、拆解、生成建议 | P0 |

## P0 能力清单

### 1. 项目知识库创建和更新

- 场景：项目立项、会后纪要沉淀、新人上手包生成
- 调用方式：`lark-cli wiki` / `lark-cli docs`
- 需要权限：知识库节点读写、云文档读写、搜索读取
- 输入：项目名称、目标、成员、节点、会议纪要、决策和风险
- 输出：项目总览页、会议记录页、详细功能设计页、变更日志
- 成功标准：项目空间结构可读、可继续深化、能被群聊主动引用
- 失败处理：如果权限不足，返回缺失 scope 和下一步授权动作
- 关联 Agent：Project Agent、Meeting Agent、Onboarding Agent

### 2. 多维表格项目作战室

- 场景：任务池、风险表、节点状态、项目看板
- 调用方式：`lark-cli base`
- 需要权限：Base 创建、表字段管理、记录读写、视图/仪表盘管理
- 输入：项目、节点、任务、风险、负责人、截止日期、状态
- 输出：项目数据库、任务池、风险表、视图和仪表盘
- 成功标准：状态可回写，任务和风险能被项目总览引用
- 失败处理：如果字段缺失，先补字段；如果记录重复，走 upsert
- 关联 Agent：Project Agent、Planner Agent、Risk Agent

### 3. 群聊消息收发

- 场景：用户触发、确认 Action Items、主动分发简报
- 调用方式：`lark-cli im`
- 需要权限：消息接收、消息发送、群聊读取
- 输入：用户文本、确认指令、项目状态变化
- 输出：群聊回复、项目简报、风险提醒、上手包
- 成功标准：能在目标群内收到消息并回复到正确上下文
- 失败处理：先确认机器人是否在群内，再确认 `send_as_user` / bot 发送权限
- 关联 Agent：Project Agent、Task Agent、Risk Agent、Onboarding Agent

### 4. 飞书任务创建和同步

- 场景：会议 Action Items、计划拆解后的执行项、逾期提醒
- 调用方式：`lark-cli task`
- 需要权限：任务创建、任务查询、任务更新、任务评论
- 输入：任务标题、负责人、截止时间、优先级、来源链接
- 输出：飞书任务 ID、任务状态、负责人确认
- 成功标准：确认后的 Action Items 能进入飞书任务并回写任务池
- 失败处理：负责人无法解析时创建待确认任务，不直接指派
- 关联 Agent：Task Agent、Meeting Agent、Planner Agent

### 5. 会议纪要和妙记读取

- 场景：会后自动总结、Action Items 提取、风险识别
- 调用方式：`lark-cli minutes` / `lark-cli vc`
- 需要权限：会议记录查询、妙记 AI 产物读取
- 输入：会议时间范围、会议 ID、妙记文档
- 输出：会议总结、章节、待办、逐字稿摘要
- 成功标准：能够把会议产物关联到项目并写入知识库
- 失败处理：无法读取妙记时允许粘贴纪要文本作为降级输入
- 关联 Agent：Meeting Agent、Risk Agent

## 能力设计原则

1. 每个能力要明确输入输出，不把业务判断藏在调用脚本里。
2. 所有写入动作都要能说明写到了哪里，以及失败后如何恢复。
3. P0 先保证可演示闭环，P1 再扩展监听、身份映射和自动化程度。
4. 任何主动分发都需要有来源链接，避免生成内容变成不可追溯的口头通知。
