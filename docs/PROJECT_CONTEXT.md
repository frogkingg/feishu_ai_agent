# 项目上下文

## 项目定位

ProjectPilot / 飞书项目领航员，是一个面向飞书协作空间的项目管理专家 Agent。

它围绕项目推进过程中的真实信息流工作：

- 群聊里的决策、分工、风险和临时讨论
- 日历里的会议、评审节点和交付节奏
- 妙记 / 会议纪要里的总结、Action Items 和阻塞
- 文档 / 知识库里的背景、需求、方案和沉淀
- 飞书任务里的执行项、负责人和截止时间
- 多维表格里的状态、进度、风险和统计

## 核心判断

这个项目不应该被拆成多个互不相关的小工具。更合理的形态是：

```text
共享底座：Project Context Pack / 项目知识包
  ├─ 场景 A：项目推进
  └─ 场景 B：新人/中途加入上手
```

两个场景使用同一套底层知识、状态和能力，只是入口和输出不同。

## P0 Demo

P0 Demo 优先验证一条闭环，而不是同时做全量监听、复杂权限和企业级项目组合管理。

建议主线：

```text
会议纪要更新项目知识包
-> 提取 Action Items
-> 创建/更新任务池
-> 更新项目作战室
-> 主动推送项目简报或新人上手包
```

## 非目标

MVP 阶段暂不做：

- 全量监听所有群聊消息
- 企业级复杂权限系统
- 完整替代飞书项目
- 跨组织协同
- 复杂甘特图和多项目组合管理
- 深度 GitHub 代码仓库分析

## 交付物

比赛交付建议保持三类：

| 交付物 | 说明 |
| --- | --- |
| 场景定义文档 | 说明为什么选择项目推进和新人上手这两个入口 |
| 可运行 Demo | 在飞书里跑通项目知识库、任务、纪要、群聊推送闭环 |
| 效果验证报告 | 说明自动维护、主动分发、任务闭环带来的效率提升 |
